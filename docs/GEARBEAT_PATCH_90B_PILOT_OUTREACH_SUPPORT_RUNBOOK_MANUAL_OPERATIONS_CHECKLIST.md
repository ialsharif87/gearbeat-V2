## 3. Support & Issue Runbook

### 3.1 Issue Severity Levels
| Level | Description | Escalation Path |
| --- | --- | --- |
| **S1 (Critical)** | Security breach, PII leak, or broken core auth. | Immediate Stop / CTO |
| **S2 (Major)** | Broken discovery flow (e.g., Studio search fails). | Technical Team |
| **S3 (Minor)** | UI typo, responsiveness glitch, or aesthetic debt. | Maintenance Patch |

### 3.2 Support Intake Workflow
1. **Intake**: Partner reports issue via `/support` or direct WhatsApp.
2. **Triage**: Support lead assigns severity level (S1-S3).
3. **Capture**: Evidence (Screenshots/Logs) added to the **Pilot Issue Tracker**.
4. **Resolution**: Fix deployed or issue logged as "Known Debt" for post-pilot hardening.
