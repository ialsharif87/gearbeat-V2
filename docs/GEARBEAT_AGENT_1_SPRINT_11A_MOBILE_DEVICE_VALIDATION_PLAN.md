# GearBeat Mobile Device Validation Plan — Sprint 11A

## 1. Objective
This plan defines the systematic approach for validating the GearBeat Mobile Mirror shell on physical hardware. It ensures the premium "app mode" experience (`?app=1`) maintains visual and functional integrity across different mobile platforms.

## 2. Testing Environments

### Local Area Network (LAN)
- **Command**: `npm run start:lan`
- **Use Case**: Primary testing within the same Wi-Fi network. Fastest refresh rates.
- **Verification**: Scanning QR code via Expo Go.

### Remote Tunneling
- **Command**: `npm run start:tunnel`
- **Use Case**: Testing over cellular data or restricted corporate networks.
- **Verification**: Stable connection over ngrok.

### Interaction
